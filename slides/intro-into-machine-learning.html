Probleem met "In-Sample" Scores

  • Berekeningen tot nu toe: "in-sample" scores
  • niet geldig zijn voor nieuwe data

Coderen

  • scikit-learn bibliotheek > data in training en validatie data te splitsen
  • `mean_absolute_error`-functie

                from sklearn.model_selection import train_test_split

                # split data into training and validation data, for both features and target
                # The split is based on a random number generator. Supplying a numeric value to
                # the random_state argument guarantees we get the same split every time we
                # run this script.
                train_X, val_X, train_y, val_y = train_test_split(X, y, random_state = 0)
                # Define model
                melbourne_model = DecisionTreeRegressor()
                # Fit model
                melbourne_model.fit(train_X, train_y)
                
                # get predicted prices on validation data
                val_predictions = melbourne_model.predict(val_X)
                print(mean_absolute_error(val_y, val_predictions))
                # 265806.91478373145

  • out-of-sample data > meer dan 250.000 dollar

Underfitting and overfitting

Experimenteer met Verschillende Modellen

Meet de nauwkeurigheid van modellen en experimenteer met alternatieve modellen om de beste voorspellingen te verkrijgen.

Diepte Boom

In de praktijk hebben bomen vaak veel splitsingen tussen de top (alle huizen) en een blad.

Overfitting

Overfitting gebeurt wanneer een model de trainingsgegevens perfect past maar slecht presteert op nieuwe gegevens.

Underfitting

Underfitting gebeurt wanneer een model belangrijke patronen in de gegevens niet vastlegt en daarom slecht presteert.

Vind de Sweet Spot

We willen het evenwicht vinden tussen underfitting en overfitting voor de beste nauwkeurigheid op nieuwe gegevens.

Vergelijk Verschillende Diepten

Max_leaf_nodes is een handige manier om overfitting te controleren. Meer bladeren verplaatsen ons naar overfitting.

Vergelijk MAE-scores

Gebruik een functie om MAE-scores te vergelijken voor verschillende max_leaf_nodes-waarden. 


from sklearn.metrics import mean_absolute_error
from sklearn.tree import DecisionTreeRegressor

def get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y):
    model = DecisionTreeRegressor(max_leaf_nodes=max_leaf_nodes, random_state=0)
    model.fit(train_X, train_y)
    preds_val = model.predict(val_X)
    mae = mean_absolute_error(val_y, preds_val)
    return(mae)
                

    # compare MAE with differing values of max_leaf_nodes
for max_leaf_nodes in [5, 50, 500, 5000]:
    my_mae = get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y)
    print("Max leaf nodes: %d  \t\t Mean Absolute Error:  %d" %(max_leaf_nodes, my_mae))
# Max leaf nodes: 5  		 Mean Absolute Error:  347380
# Max leaf nodes: 50  		 Mean Absolute Error:  258171
# Max leaf nodes: 500  		 Mean Absolute Error:  243495
# Max leaf nodes: 5000  		 Mean Absolute Error:  254983

Optimale Aantal Bladeren

Van de opties is 500 het optimale aantal bladeren.

Conclusie

Modellen kunnen lijden aan overfitting of underfitting, wat leidt tot minder nauwkeurige voorspellingen.

Inleiding

Decision trees stellen je voor een lastige keuze.

  • diepe boom > overfitting
  • ondiep > underfitting

Random forest

  • veel bomen
  • Voorspellingen worden gemiddeld over alle bomen
  • Betere voorspellende nauwkeurigheid dan enkele beslissingsboom
  • Werkt goed met standaardparameters
  • Mogelijkheid om meer modellen te leren
  • Verbeterde prestaties mogelijk, maar afhankelijk van parameters
